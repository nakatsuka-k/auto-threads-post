ブラウザ上のテーブルで管理したアカウントに対して、指定投稿を単体または全体配信します。 全体配信では時間を分散して実行できます。

アカウント管理

保存先はローカルDB（data/accounts.db）です。セッション情報は data/sessions/ に保存されます。

enabled username / ID password secret key (TOTP) session
2FA対応:
  • TOTP 2FA有効: 秘密鍵（Base32形式、例：ILOUNT6ZLNFDXN3EYB5Q4JMOXVQMZH7E）をテーブルに入力すると、ログイン時に認証コードが自動生成・入力されます。
  • 秘密鍵なし: ヘッドレス実行チェック ✗ で実行すると、ブラウザ画面が表示され、手動で2FA入力できます。セッション自動保存後は以降スキップ可能。
